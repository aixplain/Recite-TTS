# مرجع API

يوفر Action Scheduler مجموعة من الدوال لجدولة الخطافات (hooks) لتعمل في وقت ما في المستقبل لمرة واحدة أو أكثر.

لفهم دوال الجدولة، قد يساعدك التفكير فيها كامتدادات لدالة `do_action()` في WordPress تضيف القدرة على تأخير وتكرار وقت تشغيل الخطاف.

## واجهات WP-Cron مقابل واجهات Action Scheduler

صُمِّمت دوال API الخاصة بـ Action Scheduler لتعكس [دوال WP-Cron API](http://codex.wordpress.org/Category:WP-Cron_Functions) في WordPress.

تُعيد الدوال قيمًا مشابهة وتقبل وسيطات مشابهة لنظيراتها في WP-Cron. الاختلافات البارزة هي:

* `as_schedule_single_action()` و `as_schedule_recurring_action()` ستُعيدان المعرّف الخاص بالإجراء المجدول بدلاً من قيمة منطقية تشير إلى ما إذا كان الحدث قد جُدوِل
* `as_schedule_recurring_action()` تأخذ فاصلاً زمنيًا بالثواني كفاصل التكرار بدلاً من سلسلة نصية عشوائية
* `as_schedule_single_action()` و `as_schedule_recurring_action()` يمكنهما قبول معامل `$group` لتجميع إجراءات مختلفة لإضافة واحدة معًا.
* يُستبدل البادئة `wp_` بـ `as_` ويُستبدل مصطلح `event` بـ `action`

## توفر دوال API

كما ذُكر في قسم [الاستخدام - ترتيب التحميل](usage.md#load-order)، سيقوم Action Scheduler بتهيئة نفسه عند خطاف `'init'` بأولوية `1`. بينما تُحمَّل دوال API قبل ذلك ويمكن استدعاؤها، إلا أنه لا ينبغي استدعاؤها إلا بعد `'init'` بأولوية `1`، لأن كل مكوّن، مثل مخزن البيانات، لم تتم تهيئته بعد.

لا تستخدم دوال API الخاصة بـ Action Scheduler قبل خطاف `'init'` بأولوية `1`. قد يؤدي ذلك إلى نتائج غير متوقعة، مثل تخزين البيانات في موقع غير صحيح.

يوفر Action Scheduler الدالة `Action_Scheduler::is_initialized()` للاستخدام في الخطافات للتأكد من أن مخازن البيانات قد تمت تهيئتها.

## مرجع الدوال / `as_enqueue_async_action()`

### الوصف

إضافة إجراء إلى الطابور ليعمل مرة واحدة، في أقرب وقت ممكن.

### الاستخدام

```php
as_enqueue_async_action( $hook, $args, $group );
```

### المعاملات

- **$hook** (string)(مطلوب) اسم خطاف الإجراء.
- **$args** (array) الوسيطات التي تُمرَّر إلى دوال الاستدعاء عند تشغيل الخطاف. الافتراضي: _`array()`_.
- **$group** (string) المجموعة المراد تعيين هذه المهمة إليها. الافتراضي: _''_.

### القيمة المُعادة

(integer) المعرّف الخاص بالإجراء.


## مرجع الدوال / `as_schedule_single_action()`

### الوصف

جدولة إجراء ليعمل مرة واحدة في نقطة محددة في المستقبل.

### الاستخدام

```php
as_schedule_single_action( $timestamp, $hook, $args, $group );
```

### المعاملات

- **$timestamp** (integer)(مطلوب) الطابع الزمني Unix الذي يمثل التاريخ الذي تريد تشغيل الإجراء فيه.
- **$hook** (string)(مطلوب) اسم خطاف الإجراء.
- **$args** (array) الوسيطات التي تُمرَّر إلى دوال الاستدعاء عند تشغيل الخطاف. الافتراضي: _`array()`_.
- **$group** (string) المجموعة المراد تعيين هذه المهمة إليها. الافتراضي: _''_.

### القيمة المُعادة

(integer) المعرّف الخاص بالإجراء.


## مرجع الدوال / `as_schedule_recurring_action()`

### الوصف

جدولة إجراء ليعمل بشكل متكرر بفاصل زمني محدد بالثواني.

### الاستخدام

```php
as_schedule_recurring_action( $timestamp, $interval_in_seconds, $hook, $args, $group );
```

### المعاملات

- **$timestamp** (integer)(مطلوب) الطابع الزمني Unix الذي يمثل التاريخ الذي تريد تشغيل الإجراء فيه.
- **$interval_in_seconds** (integer)(مطلوب) مدة الانتظار بين عمليات التشغيل.
- **$hook** (string)(مطلوب) اسم خطاف الإجراء.
- **$args** (array) الوسيطات التي تُمرَّر إلى دوال الاستدعاء عند تشغيل الخطاف. الافتراضي: _`array()`_.
- **$group** (string) المجموعة المراد تعيين هذه المهمة إليها. الافتراضي: _''_.

### القيمة المُعادة

(integer) المعرّف الخاص بالإجراء.


## مرجع الدوال / `as_schedule_cron_action()`

### الوصف

جدولة إجراء يتكرر وفق جدول زمني شبيه بـ cron.

إذا تأخر تنفيذ إجراء شبيه بـ cron، فسيظل المحاولة التالية مجدولة وفقًا لتعبير cron المُقدَّم.

### الاستخدام

```php
as_schedule_cron_action( $timestamp, $schedule, $hook, $args, $group );
```

### المعاملات

- **$timestamp** (integer)(مطلوب) الطابع الزمني Unix الذي يمثل التاريخ الذي تريد تشغيل الإجراء فيه.
- **$schedule** (string)(مطلوب) سلسلة جدول زمني شبيهة بـ cron، انظر http://en.wikipedia.org/wiki/Cron.
- **$hook** (string)(مطلوب) اسم خطاف الإجراء.
- **$args** (array) الوسيطات التي تُمرَّر إلى دوال الاستدعاء عند تشغيل الخطاف. الافتراضي: _`array()`_.
- **$group** (string) المجموعة المراد تعيين هذه المهمة إليها. الافتراضي: _''_.

### القيمة المُعادة

(integer) المعرّف الخاص بالإجراء.


## مرجع الدوال / `as_unschedule_action()`

### الوصف

إلغاء التكرار التالي لإجراء مجدول.

### الاستخدام

```php
as_unschedule_action( $hook, $args, $group );
```

### المعاملات

- **$hook** (string)(مطلوب) اسم خطاف الإجراء.
- **$args** (array) الوسيطات المُمرَّرة إلى دوال الاستدعاء عند تشغيل الخطاف. الافتراضي: _`array()`_.
- **$group** (string) المجموعة المعيَّن إليها المهمة. الافتراضي: _''_.

### القيمة المُعادة

(null)

## مرجع الدوال / `as_unschedule_all_actions()`

### الوصف

إلغاء جميع تكرارات إجراء مجدول.

### الاستخدام

```php
as_unschedule_all_actions( $hook, $args, $group )
```

### المعاملات

- **$hook** (string)(مطلوب) اسم خطاف الإجراء.
- **$args** (array) الوسيطات المُمرَّرة إلى دوال الاستدعاء عند تشغيل الخطاف. الافتراضي: _`array()`_.
- **$group** (string) المجموعة المعيَّن إليها المهمة. الافتراضي: _''_.

### القيمة المُعادة

(string|null) المعرّف الخاص بالإجراء المجدول إذا وُجد إجراء مجدول، أو null إذا لم يُعثر على إجراء مطابق.


## مرجع الدوال / `as_next_scheduled_action()`

### الوصف

إعادة الطابع الزمني التالي لإجراء مجدول.

### الاستخدام

```php
as_next_scheduled_action( $hook, $args, $group );
```

### المعاملات

- **$hook** (string)(مطلوب) اسم خطاف الإجراء. الافتراضي: _لا يوجد_.
- **$args** (array) الوسيطات المُمرَّرة إلى دوال الاستدعاء عند تشغيل الخطاف. الافتراضي: _`array()`_.
- **$group** (string) المجموعة المعيَّن إليها المهمة. الافتراضي: _''_.

### القيمة المُعادة

(integer|boolean) الطابع الزمني للتكرار التالي، أو false إذا لم يُعثر على شيء.


## مرجع الدوال / `as_has_scheduled_action()`

### الوصف

التحقق مما إذا كان هناك إجراء مجدول في الطابور، ولكن بكفاءة أعلى من `as_next_scheduled_action()`. يُوصى باستخدام هذه الدالة عندما تحتاج إلى معرفة ما إذا كان إجراء معين مجدولًا حاليًا. _متوفرة منذ الإصدار 3.3.0._

### الاستخدام

```php
as_has_scheduled_action( $hook, $args, $group );
```

### المعاملات

- **$hook** (string)(مطلوب) اسم خطاف الإجراء. الافتراضي: _لا يوجد_.
- **$args** (array) الوسيطات المُمرَّرة إلى دوال الاستدعاء عند تشغيل الخطاف. الافتراضي: _`array()`_.
- **$group** (string) المجموعة المعيَّن إليها المهمة. الافتراضي: _''_.

### القيمة المُعادة

(boolean) true إذا كان هناك إجراء مطابق قيد الانتظار أو قيد التنفيذ، وfalse بخلاف ذلك.


## مرجع الدوال / `as_get_scheduled_actions()`

### الوصف

البحث عن إجراءات مجدولة.

### الاستخدام

```php
as_get_scheduled_actions( $args, $return_format );
```

### المعاملات

- **$args** (array) الوسيطات للبحث وتصفية النتائج. الوسيطات الممكنة مع قيمها الافتراضية:
    * `'hook' => ''` - اسم الإجراء الذي سيتم تشغيله
    * `'args' => NULL` - مصفوفة الوسيطات التي ستُمرَّر مع الإجراء
    * `'date' => NULL` - تاريخ جدولة الإجراء. يتوقع كائن DateTime، أو طابع زمني Unix، أو سلسلة نصية يمكن تحليلها بـ strtotime().
    * `'date_compare' => '<=`' - عامل المقارنة لاختبار "date". القيم المقبولة هي '!='، '>'، '>='، '<'، '<='، '='
    * `'modified' => NULL` - تاريخ آخر تحديث للإجراء. يتوقع كائن DateTime، أو طابع زمني Unix، أو سلسلة نصية يمكن تحليلها بـ strtotime().
    * `'modified_compare' => '<='` - عامل المقارنة لاختبار "modified". القيم المقبولة هي '!='، '>'، '>='، '<'، '<='، '='
    * `'group' => ''` - المجموعة التي ينتمي إليها الإجراء
    * `'status' => ''` - ActionScheduler_Store::STATUS_COMPLETE أو ActionScheduler_Store::STATUS_PENDING
    * `'claimed' => NULL` - TRUE للبحث عن إجراءات مطالَب بها، FALSE للبحث عن إجراءات غير مطالَب بها، أو سلسلة نصية للبحث عن معرّف مطالبة محدد
    * `'per_page' => 5` - عدد النتائج المراد إعادتها
    * `'offset' => 0`
    * `'orderby' => 'date'` - القيم المقبولة هي 'hook'، 'group'، 'modified'، أو 'date'
    * `'order' => 'ASC'`
- **$return_format** (string) الصيغة التي تُعاد بها الإجراءات المجدولة: 'OBJECT'، 'ARRAY_A'، أو 'ids'. الافتراضي: _'OBJECT'_.

### القيمة المُعادة

(array) مصفوفة من صفوف الإجراءات المطابقة للمعايير المحددة في `$args`.
